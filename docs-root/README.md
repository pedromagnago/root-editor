# docs-root — documentação da Operação Root

Metodologia, PRD, arquitetura e decisões da **Máquina de Carrossel V03**. Isto é
documentação da Root, **não** do Open Design (os docs do upstream ficam em
`../docs/`). Vive aqui para viajar junto com o código no `git pull` do time.

- `00-VISAO-GERAL.md` — o produto em uma página.
- `00-fundamentos/` — contexto fixo (metodologia, princípios, commits, observabilidade).
- `01-prd.md` … `08-layout-claude-code.md` — as 6 camadas do kit de metodologia.
- `09-correcao-identidade-de-marca.md` — auditoria "por que todo carrossel sai com a cara da Root" + as 5 ondas de correção (20/07/2026).

> **Fonte da verdade da metodologia editorial é a skill**, em `../skills/carrossel-root/`
> (`SKILL.md` + `knowledge/`). Estes docs registram o *porquê* das decisões; a skill
> é o *como* que roda. Editar a metodologia = editar a skill, nunca só o doc.
